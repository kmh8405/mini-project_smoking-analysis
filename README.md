## 2️⃣ 전처리 (Preprocessing)

전처리는 단순한 데이터 정리가 아니라, **분석 결과의 신뢰도를 결정하는 핵심 단계**라고 판단하고 설계했습니다.

특히 각 처리 방식에 대해 “왜 이 방법을 선택했는지”를 기준으로 접근했습니다.

---

### ✔️ 1. 결측치 처리 전략

#### 📌 시력 → 최빈값 (mode)

시력 데이터는 다음과 같은 특징을 가짐:

- 이산적인 값 (0.1 단위)
- 특정 값(1.0 등)에 데이터가 집중됨
- 평균/중앙값 사용 시 실제 존재하지 않는 값 생성 가능

👉 따라서 **가장 많이 등장하는 값(최빈값)이 실제 데이터를 가장 잘 대표한다고 판단**

---

#### 📌 공복 혈당 / 혈압 / 중성 지방 → 중앙값 (median)

이 변수들은 공통적으로:

- 이상치(outlier)의 영향을 많이 받는 연속형 변수
- 실제 분포가 오른쪽으로 치우친(skewed) 형태

👉 평균(mean)을 사용할 경우, 극단값에 의해 대표값이 왜곡될 가능성이 큼

👉 따라서 **이상치에 강건한 중앙값을 사용**

---

### ✔️ 2. 도메인 기반 이상치 처리

단순 통계 기준(IQR)만으로는 **“물리적으로 불가능한 값”을 제거할 수 없다고 판단**

---

#### 📌 시력

- 일반적인 시력 범위: 0 ~ 2.5
- 데이터 내 비정상 값 존재 가능

👉 clip(0, 2.5) 적용

---

#### 📌 혈압

- 30 미만 → 생리적으로 불가능한 수준

👉 해당 값 NaN 처리 후 중앙값 보정

---

#### 📌 몸무게

- 성인 기준 40kg 미만 → 극히 비현실적인 값

👉 데이터 오류 가능성으로 판단 후 제거

---

#### 📌 BMI

- 극단값은 존재 가능하지만 분석 안정성을 해칠 수 있음

👉 15 ~ 45 범위를 기준으로 이상치 제거 후 보정

---

### ✔️ 3. 통계 기반 이상치 처리 (IQR)

다음 변수들은:

- 물리적으로 완전히 불가능한 값이라기보다는
- **분포 상 극단값이 존재하는 경우**

적용 변수:
- 혈청 크레아티닌
- 저밀도지단백
- 헤모글로빈

👉 IQR 기준으로 clipping 적용

---

### ✔️ 4. 분포 안정화 (로그 변환)

#### 📌 중성 지방 → log 변환

EDA에서 확인된 특징:

- 오른쪽으로 긴 꼬리(long tail)
- 고값(outlier) 존재
- 정규분포와 거리 있음

👉 이러한 분포에서는

- 평균 기반 분석(t-test)
- 상관 분석

의 신뢰도가 떨어질 수 있음

---

👉 log 변환 적용 효과:

- 분포 왜곡 완화
- 극단값 영향 감소
- 통계 분석 안정성 증가

---

### ⚠️ 전처리 순서에 대한 시행착오

초기에는 다음과 같은 흐름으로 진행했습니다:

```text
결측치 처리 → 이상치 처리
```

하지만 이 방식은 이상치가 포함된 상태에서 중앙값이 계산되는 문제가 있었습니다.

이후 다음과 같은 접근을 고민했습니다:

```text
이상치 → 결측치
```

그러나 도메인 기반 처리와 결측치 재보정 로직이 함께 섞이면서
오히려 코드 복잡도가 증가하는 문제가 발생했습니다.

---

### ✅ 최종 결정

결측치 처리 → 도메인 기반 이상치 처리 → 통계 기반 보정(IQR)의 순서로 적용하였다.

이는 결측값을 먼저 안정적으로 보정한 뒤, 도메인 기준으로 비정상 값을 제거하고, 마지막으로 통계적 기준으로 분포를 정리하기 위함이다.

👉 단순 순서 문제가 아니라 **“각 처리의 목적을 분리하는 방향”으로 구조 개선**

## 3️⃣ 가설 수립 및 검정

EDA 결과를 기반으로 총 4개의 가설을 수립하고 검증했습니다.

---

### 📌 가설 1: 헤모글로빈

- H₀: 흡연자와 비흡연자의 평균 헤모글로빈은 차이가 없다
- H₁: 차이가 있다

👉 검정: t-test

**결과**
- p-value < 0.05

👉 **귀무가설 기각**

**해석**
- 흡연 여부에 따라 헤모글로빈 수치 차이가 존재

---

### 📌 가설 2: 중성지방 (순위 관계)

- H₀: 흡연 여부와 중성지방 간 상관관계 없음
- H₁: 상관관계 있음

👉 검정: Spearman

**결과**
- p-value < 0.05

👉 **귀무가설 기각**

**해석**
- 흡연 여부와 중성지방 사이에 유의미한 관계 존재

---

### 📌 가설 3: 비만 집단 내 차이

- H₀: 비만 집단에서 흡연자/비흡연자 간 중성지방 차이 없음
- H₁: 차이 있음

👉 검정: t-test

**결과**
- p-value < 0.05

👉 **귀무가설 기각**

**해석**
- 단순 흡연 여부뿐 아니라 **비만 상태에 따라 관계가 더 강화됨**

---

### 📌 가설 4: 충치

- H₀: 흡연 여부와 충치는 독립
- H₁: 독립이 아님

👉 검정: Chi-square

**결과**
- p-value < 0.05

👉 **귀무가설 기각**

**해석**
- 흡연 여부와 충치 발생 간 연관성 존재

### 4. 부동소수점 오차 및 데이터 표현 문제

전처리 이후 CSV 파일을 확인하는 과정에서 일부 변수에서 다음과 같은 값이 발견되었습니다.

```text
1.2999999999998
```

이는 Python의 부동소수점 연산 특성으로 인해 발생하는 문제로, 실제 값이 잘못된 것이 아니라 **표현상의 오차**입니다.

#### 문제점

- 데이터 가독성 저하
- 시각화 시 비정상적인 값처럼 보일 수 있음
- 보고서 및 결과 해석 신뢰도 저하

#### 해결 방법

변수의 특성과 도메인 기준에 맞춰 소수점 자리수를 정리하였습니다.

- 혈청 크레아티닌 → 소수 2자리
- 시력 → 소수 2자리
- 중성 지방_log → 소수 3자리

또한, 로그 변환 이후에 수치 정리를 수행하도록  
전처리 순서를 다음과 같이 수정했습니다.

- log_transform → clean_numeric_format

이를 통해 데이터의 **가독성과 분석 신뢰도를 동시에 개선**할 수 있었습니다.
